Lenguaje Dart
Framework Flutter
proyecto Papeleria 
Actúa como un arquitecto de software senior especializado en Flutter, Dart y Firebase. Necesito que desarrolles un plan de implementación profesional y detallado en formato Markdown para crear una aplicación multiplataforma de gestión de papelería compatible con Android, Web y Windows, utilizando Flutter, Firebase y Visual Studio Code o Antigravity como entorno de desarrollo.
El objetivo es construir un sistema moderno, escalable, modular y fácil de mantener, pensado para automatizar y administrar los procesos principales de una papelería, incluyendo control de inventario, ventas, compras, clientes, proveedores, caja y administración de usuarios. La aplicación debe estar preparada para crecimiento futuro, múltiples sucursales y posibles nuevas funcionalidades, por lo que necesito una arquitectura bien organizada y orientada a buenas prácticas profesionales.
La aplicación deberá utilizar navegación mediante rutas nombradas desde main.dart, manejo de estado con Provider, autenticación segura mediante Firebase Authentication y almacenamiento de información en Cloud Firestore. También debe contemplar el uso de Firebase Storage para imágenes de productos y recursos multimedia. Quiero que el sistema tenga una estructura limpia y profesional, separando correctamente modelos, pantallas, servicios, providers, widgets reutilizables, repositorios, utilidades y configuración general del proyecto.
Necesito que el plan explique detalladamente cómo organizar la arquitectura del proyecto, cómo estructurar las carpetas y cuál será el flujo recomendado de desarrollo desde la configuración inicial hasta el despliegue final. También quiero recomendaciones sobre dependencias necesarias para pubspec.yaml, herramientas complementarias, estrategias de seguridad, validaciones, control de sesiones y manejo de roles y permisos para diferentes tipos de usuarios como administrador, empleado o cajero.
El sistema deberá incluir módulos CRUD completos para productos, categorías, proveedores, clientes, empleados, ventas, compras, movimientos de inventario, pagos, cuentas por cobrar, cuentas por pagar, sucursales, cajas y permisos de usuario. Además, quiero que se expliquen las relaciones entre entidades en Firestore y la mejor manera de estructurar la base de datos para mantener rendimiento, escalabilidad y facilidad de mantenimiento.
También necesito que el plan contemple aspectos de UI/UX, incluyendo diseño moderno, interfaz responsive para distintas plataformas, navegación intuitiva, reutilización de componentes visuales y recomendaciones para una experiencia de usuario eficiente tanto para administradores como para cajeros.
Finalmente, quiero un roadmap completo de desarrollo dividido por fases, indicando el orden correcto de implementación, desde la configuración del entorno y Firebase, autenticación, diseño de base de datos y desarrollo de módulos CRUD, hasta testing, optimización y despliegue multiplataforma.
No quiero código todavía. Quiero únicamente un plan técnico profesional, detallado y bien estructurado para comenzar correctamente el desarrollo del sistema de papelería.
